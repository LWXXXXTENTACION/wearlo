# Loop 强制约束

以下规则适用于每次自动或辅助闭环运行。

## 必须执行

- 修改前阅读根目录 `AGENTS.md` 和 `docs/agent/` 下的任务相关配置。
- 保留已有工作区改动，每轮只处理一个任务。
- 使用 `@github` 前从本地 remote 或用户输入确认 `owner/repo`，不得猜测目标仓库。
- 报告完成前运行 `docs/agent/HARNESS.md` 中所有适用的最终门禁。
- 三次尝试仍未成功时停止，并附证据说明阻塞原因。

## 严格禁止

- 闭环运行期间修改 `.env`、`.env.*`、`.git/` 或闭环控制文件。
- 暴露秘密信息、削弱认证或权限检查、禁用测试或静态检查。
- 未经明确授权，通过 `@github` 创建或修改 Issue、评论、标签、PR、分支或仓库内容。
- 未经批准安装依赖、访问生产服务、提交、推送、合并、部署或删除用户数据。
- 使用破坏性 Git 命令，或丢弃、覆盖无关改动。

## 必须升级人工处理

需求含糊、必须改变数据结构或认证契约、需要新增依赖、验证依赖外部凭据，或受保护路径需要变更时，暂停并请求人工决定。
